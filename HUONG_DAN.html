FL-SMPC++ — Tái dựng theo Hình 3

Hệ thống được chia thành 3 thư mục rõ ràng: dataset, client, server — để dễ hiểu và dễ chỉnh sửa. Có 2 cách chạy: mô phỏng 1 máy (nhanh, để hiểu logic) hoặc triển khai thật qua HTTP trên nhiều máy.

Cách 1 — Mô phỏng nhanh trên 1 máy:

python chay.py

Cần numpy và scikit-learn. Chạy khoảng vài giây, in ra từng vòng huấn luyện. Đây là cách nhanh nhất để xem toàn bộ cơ chế hoạt động — mọi "client" chỉ là object Python chạy tuần tự trong cùng 1 tiến trình.

Cách 2 — Triển khai thật trên nhiều máy (HTTP): xem mục 7 bên dưới. Dùng khi có nhiều máy/máy ảo đóng vai client thật, kết nối tới 1 máy server thật qua mạng.

2. Luồng hoạt động (Hình 3)

Mỗi vòng huấn luyện, dữ liệu chạy từ client sang server theo luồng sau — đúng cho cả 2 cách chạy (mô phỏng hay HTTP thật), chỉ khác cách gói dữ liệu di chuyển (gọi hàm trực tiếp trong RAM, hay gửi qua mạng):

PHÍA CLIENT (mỗi client tự làm)

  1. Huấn luyện cục bộ trên dữ liệu riêng → Δw
  2. (nếu độc hại) bóp méo Δw
  3. Niêm phong bằng Pedersen → C
  4. Chấm điểm trên validation chung → S
  5. Sinh chứng minh ZKP
  6. Che bằng SMPC → Δw̃

Gửi gói (S, C, ZKP, Δw̃) ↓

PHÍA SERVER (aggregator)

  1. Cửa 1: xác minh ZKP
  2. Cửa 2: đối chiếu vân tay commitment
  3. Cửa 3: lọc theo điểm validation S ≥ θ
  4. Kiểm quorum (đủ client hợp lệ?)
  5. Tổng hợp SMPC (nhiễu triệt tiêu)
  6. Cập nhật mô hình toàn cục

4. Cách chỉnh sửa thường gặp

Đổi tham số (dễ nhất — không đụng code logic)

Mở cau_hinh.py, sửa giá trị rồi chạy lại. Ví dụ:

SO_CLIENT = 20            # tăng lên 20 client
CLIENT_DOC_HAI = [0,1,2,3,4,5]  # 30% của 20 = 6 kẻ độc hại
NGUONG_VALIDATION = 0.6   # siết ngưỡng lọc
NON_IID = False           # đổi sang chia đều (IID)

Đổi sang dữ liệu thật (MNIST, CIFAR...)

Mở dataset/du_lieu.py, sửa duy nhất hàm _nap_goc(). Xem hướng dẫn chi tiết trong dataset/README.md.

Đổi mô hình sang CNN

Mở client/mo_hinh.py. Chỉ cần giữ nguyên "giao diện": hàm huan_luyen_cuc_bo(...) phải trả về véc-tơ phẳng Δw. Phần mật mã và SMPC không cần đổi.

Thêm kiểu tấn công mới

Mở client/client.py, trong phương thức chay_mot_vong() thêm nhánh xử lý cho kiểu tấn công của bạn (hiện có sẵn dao_nhan).

5. Đọc kết quả

Mỗi dòng khi chạy có dạng:

Vòng  5 | nhận 7/10 client | loại 3/3 kẻ độc hại | test acc = 88.9%
  • nhận 7/10: server chấp nhận 7 trong 10 client (3 kẻ độc hại bị loại)
  • loại 3/3 kẻ độc hại: cả 3 kẻ đảo nhãn đều bị chặn
  • test acc: độ chính xác mô hình toàn cục trên tập kiểm tra
Có thể bạn sẽ thấy: ở các vòng sau, cột "loại" tụt từ 3/3 xuống 2/3 hoặc 1/3 — một kẻ gian lọt qua. Đây không phải lỗi. Khi mô hình toàn cục mạnh dần, mô hình đảo nhãn của kẻ gian cũng đạt điểm validation cao hơn và vượt ngưỡng cố định θ=0.55. Đây chính là điểm yếu của ngưỡng tuyệt đối mà bài báo dùng.
Lưu ý trung thực về Cửa 2: khi client che Δw bằng SMPC, vân tay của bản đã che khác vân tay bản gốc đã cam kết — nên Cửa 2 (đối chiếu vân tay) không dùng được nguyên trạng. Đây là một khoảng hở thật trong thiết kế bài báo: ZKP chứng minh "biết cách mở commitment" chứ không chứng minh "bản đã che chính là bản đã cam kết". Xem server/README.md để hiểu rõ hơn. Trong bản demo, Cửa 1 (ZKP) và Cửa 3 (validation) gánh phần lọc chính.

7. Triển khai thật trên nhiều máy (HTTP)

chay.py chỉ mô phỏng trong 1 tiến trình — mọi "client" thật ra là object Python chạy tuần tự, không có mạng thật. Để chạy trên các máy/máy ảo riêng biệt (ví dụ 5 máy ảo đóng vai client, 1 máy thật làm server), dùng bộ 4 file: chuan_bi_du_lieu.py, server_http.py, client_http.py, giao_tiep.py.

Nguyên tắc quan trọng: dữ liệu không rời khỏi máy sở hữu nó

Khác với bản mô phỏng (server giữ sẵn dữ liệu của mọi client trong RAM), bản HTTP tách bạch rõ:

  • Server chỉ giữ tập warm-up + test (dùng nội bộ) và tập validation chung (công khai, ai cũng thấy giống nhau).
  • Mỗi client giữ dữ liệu riêng của mình trong 1 file .npz ngay trên máy đó — không có API nào để tải dữ liệu riêng của client qua mạng.

Các bước triển khai

  1. Chuẩn bị dữ liệu (chạy 1 lần, ở đâu cũng được): 
    python chuan_bi_du_lieu.py
    Sinh ra: du_lieu_server.npz, du_lieu_validation.npz (giữ ở máy server) và du_lieu_client_0.npz … du_lieu_client_{N-1}.npz (mỗi file copy sang đúng 1 máy client, rồi xoá khỏi máy chuẩn bị).
  2. Đổi SO_CLIENT trong cau_hinh.py cho khớp số máy ảo thật (ví dụ 5), rồi chạy lại bước 1.
  3. Trên máy SERVER: 
    pip install flask numpy scikit-learn
python server_http.py
    Server lắng nghe cổng 5000, in ra địa chỉ IP cần dùng.
  4. Copy sang mỗi máy ẢO client: thư mục client/, file nen_mat_ma.py, giao_tiep.py, client_http.py, và ĐÚNG 1 file du_lieu_client_<id>.npz của máy đó.
  5. Trên mỗi máy ẢO client: 
    pip install requests numpy
python client_http.py <id> http://<IP-máy-server>:5000
    Ví dụ máy ảo số 3: python client_http.py 3 http://192.168.1.10:5000

Luồng hoạt động qua mạng

BướcAi gọiEndpointÝ nghĩa
1Client (lúc khởi động)GET /cau-hinhlấy tham số huấn luyện + biết mình có phải client độc hại không
2Client (lúc khởi động)GET /du-lieu-validationlấy tập validation chung (công khai)
3Client (mỗi vòng)GET /mo-hinhlấy mô hình toàn cục hiện tại + số vòng
4Client (mỗi vòng)POST /goigửi gói (S, C, proof, Δw̃) — request này CHỜ tới khi đủ client hoặc hết giờ, rồi nhận về mô hình mới

Server dùng threading.Condition để đồng bộ: khi đủ SO_CLIENT gói (hoặc hết TIMEOUT_VONG giây), server tự lọc 3 cửa + tổng hợp SMPC + cập nhật mô hình, rồi trả kết quả cho MỌI client đang chờ cùng lúc.

Đã kiểm thử: chạy 10 tiến trình client_http.py thật (qua HTTP, đọc dữ liệu từ file .npz cục bộ) chống lại 1 server_http.py thật trên localhost — kết quả accuracy từng vòng khớp với bản mô phỏng chay.py (~87-89% sau 15 vòng).
Về bảo mật mạng: server_http.py dùng HTTP thường (không TLS) — phù hợp cho demo/lab trong mạng nội bộ tin cậy (VD các máy ảo cùng 1 mạng riêng). Nếu triển khai qua mạng không tin cậy, cần thêm HTTPS và giới hạn firewall chỉ cho phép IP client đã biết kết nối tới cổng server.

Bản tái dựng dùng mô hình softmax thay CNN và bộ dữ liệu digits thay MNIST/CIFAR để chạy nhanh không cần GPU. Mọi cơ chế bảo mật (Pedersen, ZKP, SMPC, lọc) đều chạy thật. Con số tuyệt đối khác bài báo; xu hướng so sánh thì giữ nguyên. Xem sodo.html để có sơ đồ trực quan cả 2 cách triển khai.
