Sơ đồ triển khai — FL-SMPC++

Có 2 cách triển khai, tách thành 2 thư mục riêng: mo_phong/ (mô phỏng nhanh trong 1 tiến trình) và trien_khai_http/ (chạy thật qua HTTP trên nhiều máy/máy ảo riêng biệt). Sơ đồ dưới đây thể hiện thứ tự thực thi file cho cả 2 cách.

Dùng chung / hạ tầng triển khai dataset/ client/ server/
CÁCH 1

1. Mô phỏng 1 máy — python mo_phong/chay.py

Toàn bộ chạy trong 1 tiến trình duy nhất: mỗi "client" chỉ là 1 object Python, không có mạng thật. Dùng để hiểu nhanh logic hệ thống.

CÁCH 2

2. Triển khai thật qua HTTP — nhiều máy riêng biệt

Dùng khi có nhiều máy/máy ảo đóng vai client thật (ví dụ 5 máy) kết nối tới 1 máy server thật qua mạng. Dữ liệu riêng của mỗi client không rời khỏi máy đó — chỉ mô hình + gói (S, C, proof, Δw̃) đi qua mạng.

2.1. Chuẩn bị dữ liệu (chạy 1 lần)

A trien_khai_http/chuan_bi_du_lieu.py 1 lần duy nhất
gọi dataset/du_lieu.py::nap_va_chia_du_lieu() rồi tách ra thành file .npz (ở thư mục gốc dự án)
  • du_lieu_server.npz (warm-up + test) → giữ ở máy SERVER
  • du_lieu_validation.npz (validation chung) → giữ ở máy SERVER
  • du_lieu_client_0.npz … du_lieu_client_N.npz → mỗi file COPY tay sang đúng 1 máy client, rồi XOÁ khỏi nơi khác

2.2. Sơ đồ 2 máy — server thật ↔ máy ảo client

MÁY SERVER THẬT trien_khai_http/server_http.py
nạp du_lieu_server.npz + du_lieu_validation.npz — KHÔNG có dữ liệu riêng của client nào
  • tạo Server (dùng server/server.py) + warm-up
  • GET /cau-hinh — tham số + ai là client độc hại
  • GET /du-lieu-validation — tập chung công khai
  • GET /mo-hinh — mô hình toàn cục + số vòng hiện tại
  • POST /goi — nhận gói, CHỜ đủ client hoặc hết giờ → lọc 3 cửa → tổng hợp SMPC → cập nhật mô hình → trả kết quả cho mọi client đang chờ
MÁY ẢO CLIENT #i trien_khai_http/client_http.py <i>
chỉ có: client/, nen_mat_ma.py, trien_khai_http/ (client_http.py + giao_tiep.py), du_lieu_client_i.npz
  • đọc dữ liệu riêng từ du_lieu_client_i.npz (file cục bộ, KHÔNG qua mạng)
  • tạo Client (dùng client/client.py)
  • vòng lặp: chay_mot_vong() → POST /goi → nhận mô hình mới
HTTP (JSON qua giao_tiep.py)
giao_tiep.py chuyển đổi 2 chiều: điểm đường cong EC (tuple số nguyên lớn) ↔ string, mảng numpy (Δw̃, mô hình) ↔ list JSON — để gói dữ liệu mật mã có thể đi qua mạng HTTP bình thường.

Cách đọc sơ đồ: khối màu vàng nâu (dùng chung/hạ tầng triển khai) không phụ thuộc thư mục nào. Khối xám (dataset) chỉ chạy ở nơi có toàn quyền truy cập dữ liệu gốc. Khối xanh (client) và khối cam (server) ở Cách 1 nằm chung 1 vòng lặp trong RAM; ở Cách 2 chúng tách thành 2 tiến trình/máy riêng, nói chuyện qua HTTP với khung nét đứt ngăn cách bằng mạng thật.
